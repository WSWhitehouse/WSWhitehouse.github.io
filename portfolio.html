Pong in C++

As part of my university course at UWE, I created a Pong clone in C++. It was created in a bespoke OpenGL engine called ASGE. It is specially designed and targeted for students as it has bare-bones functionality and provides minimum hand-holding.
This small Pong clone was an important stepping stone, as it allowed me to get familiar with some new (at the time) programming concepts, such as pointers, memory management and classes. The game made use of vectors and points to store the location of objects, and used vector math to move the ball and paddles on-screen. Each object had an Axis-Aligned Bounding Box (AABB) which means I used AABB-AABB collision detection to detect collisions between the ball and paddles.

Angry Birds in C++

The Angry Birds game was my final first year submission to one of my university modules. It was developed in the ASGE engine, and I followed a component-based design pattern. This design pattern splits each individual function or game logic into its own component class. These components can then be added, removed and reused on any object. Using this design pattern allows me to display an understanding of virtual/abstract classes, class inheritance, funtion overriding and how to structure my code cleanly.


Creating a 3D Environment in Maya & Unity

In Maya I created a carriage that moves over a large valley. The carriage moves using a mechanical driving mechanism, in this case: gears, gear racks, and a crank. Half way across the valley, some part of the mechanism breaks, then get fixed and the carriage will proceed to the other side. Throughout creating this 3D environment, I learnt how to model 3D objects, create materials, apply materials and textures to custom 3D models, setup scene lighting and animate. This was a submission for university and the 3D environment had a maximum tri count of 50,000.
The Unity project containted the Maya 3D environment, I created a third person character controller using Unity's animation system with a fully dynamic camera system. The player controls the chartacter controller and they must reach the other side of the valley. The player can interact with certain objects in the scene, such as the crank or the carriage, this is done through raycasting. The dynamic camera system will change which camera is best to use to give the player the best view. The camera that is attached to the player does not clip into the environment and objects can not cut it off, this is done by firing a raycast from the camera to the player. If an object is in the way, the camera will move in-front of the object so the player can still see the character. Once the player reaches the other side, the player must shoot a gun and destroy some "space ships". A particle system was used for the gun charge and shot.
